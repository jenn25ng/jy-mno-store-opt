# 매장수 최적화 서비스 기획서

> SKT 오프라인 매장 포트폴리오 최적화 — 분석 방법론 및 서비스 설계

---

## 1. 문제 정의

### 배경
- SKT 오프라인 매장을 최적화해서 운영하는 방향으로 전략 전환 필요
- 단순 비용 절감이 아닌 **M/S 방어 + 수익성 개선 + 커버리지 유지**를 동시에 달성해야 함
- 의사결정을 데이터/AI 기반으로 전환하고자 함

### 핵심 질문
> **"어떤 매장을 닫아도 SKT 점유율과 고객 경험에 타격이 없는가?"**

### 목적함수
매장 폐점 결정은 단일 지표로 환원 불가. 아래 3가지 렌즈를 동시에 평가:

| 렌즈 | 핵심 질문 | 판단 기준 |
|---|---|---|
| **M/S 방어** | 닫아도 시장점유율이 유지되는가? | 인근 SKT 매장으로 가입 흡수 가능 여부 |
| **수익성** | 운영비 대비 창출 가치가 적자인가? | 비용 대비 LTV·순증 기여도 |
| **커버리지** | 해당 상권의 대체 접점이 있는가? | 반경 내 대체 매장 수·유동인구 커버 여부 |

3개 렌즈 모두 "닫아도 된다" → **정리 우선 후보**
1~2개 걸림 → **조건부 / 유지 권고**

## 2. 사용자 및 요구사항

### 사용자
| 구분 | 역할 | 주요 관심사 |
|---|---|---|
| 유통혁신팀 | 전국 포트폴리오 조정 결정 | 전국 현황 한눈에, 정리 후보 목록, 시뮬레이션 결과 |
| 지역본부 | 권역 내 매장 효율 관리 | 내 권역 지도 보기, 특정 매장 상세 분석 |

### 핵심 요구사항
1. AI가 전국 매장에 대해 **정리 우선순위 스코어**를 자동 산출하고 순위를 보여줄 것
2. 특정 매장 선택 시 **3개 렌즈별 평가 + AI 종합 의견** 제공
3. **"이 매장을 닫으면?"** 시뮬레이션: M/S 변화·커버리지 공백·흡수 가능 매장 자동 계산
4. 경쟁사(KT/LGU+) 매장 위치를 지도에 오버레이하여 상권 공백 파악
5. 본사와 현장이 각자 편한 진입점(리스트 vs 지도)으로 접근 가능

## 4. 분석 방법론

### 4-0. 설계 철학

> **"AI는 신호를 주고, 사람이 판단한다"**

매장 폐점에는 정해진 정답이 없음. 인구수 대비, LTV 관점, 경쟁 대응 관점 등 어느 하나로 환원되지 않으며, 전체 최적화보다 **"여긴 줄여야 / 여긴 생겨야"의 지역별 신호 합**으로 접근하는 것이 현실적.

따라서 이 시스템은:
- **알고리즘이 "폐점하라"고 결론 내리지 않음**
- 3개 분석 레이어가 독립적·병렬적으로 각자 신호를 생성
- 렌즈 판정 단계에서 목적함수(3개)별 ✓/✗를 집계
- 사람(유통혁신팀, 지역본부)이 신호를 보고 최종 판단

가중 스코어링(weighted scoring) 방식은 직관적이지만 가중치 자체가 임의값이라는 근본적 한계가 있어 **주된 방법론으로 채택하지 않음.**

---

### 4-1. 전체 파이프라인

```
사내 DB (순증/LTV/운영비/M/S/상권)
           +
경쟁사 데이터 (SKT 1,275 / KT 134 / LGU+ 852개)
                      ↓
        ┌─────────────┼─────────────┐
        │             │             │
  [레이어 A]    [레이어 B]    [레이어 C]
   DBSCAN      Iso Forest     Huff 모델
  클러스터      수익성 이상   수요 이동
   밀도 계산    패턴 탐지    시뮬레이션
        │             │             │
        └─────────────┼─────────────┘
                      ↓
              [렌즈 판정 단계]
         수익성 ✓/✗ | M/S방어 ✓/✗ | 커버리지 ✓/✗
            (임계값은 의사결정자가 설정)
                      ↓
              [Claude API]
              자연어 해석 + 인사이트
                      ↓
              사람이 최종 판단
          (유통혁신팀 / 지역본부장)
```

### 4-2. 분석 레이어 (병렬 실행)

3개 레이어는 순차적으로 흐르는 게 아니라 **독립적으로 각자 신호를 계산**함.
레이어 간 선후관계 없음 — 각 레이어가 다른 각도에서 매장을 진단.

**레이어 A — DBSCAN (지리적 밀집 탐지)**

DBSCAN(Density-Based Spatial Clustering)으로 인근에 매장이 밀집된 구역을 자동 군집화.
사람이 기준을 정하지 않아도 데이터 분포에서 과밀 구역이 드러남.

```
입력: 전국 SKT 매장 위경도
출력: 클러스터 그룹 (밀집 구역) + 고립 매장 (단독 입지)
      → 클러스터 내 매장 수(cluster_size), 클러스터 ID
```

- 반경(ε): 도심 0.8km / 수도권 1.5km / 지방 3.0km (상권 유형별 차등)
- 최소 매장 수(min_samples): 2개 이상이면 클러스터로 인정

*→ 커버리지 렌즈 판정에 사용 (cluster_size ≥ 임계값이면 대체 매장 있음)*
*리서치 ① 잠식 분석과 연계*

**레이어 B — Isolation Forest (수익성 이상 탐지)**

"폐점해야 한다"는 정답 레이블 없이, 전체 매장 중 수익성 패턴이 다른 매장을 탐지.

```
입력 피처: 월 순증 / LTV / 월 운영비 / 유동인구 / 채널 기능 유형 / M/S
출력: anomaly score (낮을수록 이상) + 이상 여부 플래그
```

이상치 ≠ 폐점 후보. "주목해야 할 매장"이며 이유는 다양함:

| 이상치 유형 | 실제 의미 | 판단 방향 |
|---|---|---|
| 운영비 높음 + 순증 낮음 + 인근 SKT 多 | 잠식 받는 비효율 매장 | 정리 검토 |
| 경쟁사 밀집 속 유일한 SKT 매장 | 전략적 요충지 | 유지 필수 |
| 출점 3개월 미만 | 데이터 미성숙 | 판단 보류 |
| 도서·산간 단독 입지 | 커버리지 유지 목적 | 유지 필수 |

*→ 수익성 렌즈 판정에 사용 (손익 < 0 AND if_score < 임계값)*
*리서치 ③ SKT CV센터 선례·④ 채널 기능 분류와 연계*

**레이어 C — Huff 모델 (수요 이동 시뮬레이션)**

폐점 시 해당 매장 수요가 인근 SKT/경쟁사 매장으로 어떻게 이동하는지 확률적으로 계산.
레이어 A·B 결과와 무관하게 **전체 SKT 매장** 대상으로 실행.

```
P(고객 i → 매장 j) = (매장규모_j / 거리_ij²) ÷ Σ(매장규모_k / 거리_ik²)

출력: 흡수율 (인근 SKT 매장이 수요를 가져가는 비율)
      이탈 추정 (경쟁사로 넘어갈 고객 수)
      커버리지 공백 (폐점 후 SKT 접점 없어지는 유동인구)
```

*→ M/S 방어 렌즈 판정에 사용 (흡수율 ≥ 임계값 AND 인근 SKT 수 ≥ 임계값)*
*리서치 ② 커버리지 공백·화이트존 동시 분석과 연계*

### 4-3. 렌즈 판정 — 목적함수 ✓/✗ 집계

3개 레이어 출력값을 목적함수(3개 렌즈)별로 판정하는 단계.
**임계값은 의사결정자가 직접 설정하거나 시뮬레이션으로 조정 가능.**

| 렌즈 | 판정 조건 | 사용 레이어 출력 | 기본 임계값 |
|---|---|---|---|
| **수익성** | 적자 운영 중 AND 패턴 이탈 | Iso Forest (if_score, 월손익) | 손익 < 0, if_score < -0.05 |
| **M/S 방어** | 인근 SKT가 충분히 흡수 가능 | Huff (흡수율, 인근 SKT 수) | 흡수율 ≥ 70%, 인근 SKT ≥ 2개 |
| **커버리지** | 같은 클러스터 내 대체 매장 충분 | DBSCAN (cluster_size) | 클러스터 내 매장 ≥ 3개 |

판정 결과:

| 통과 렌즈 수 | 판정 |
|---|---|
| 3개 모두 ✓ | 정리 우선 |
| 2개 ✓ | 조건부 검토 |
| 1개 ✓ | 관찰 |
| 0개 | 유지 |

임계값을 바꾸면 판정 결과가 달라짐 — 의사결정자가 "얼마나 엄격하게 볼 것인가"를 직접 조정할 수 있음.

---

### 4-4. 자연어 설명 (Claude API)

**목적**: 렌즈 판정 결과를 의사결정자가 읽을 수 있는 언어로 변환

```
입력: 3개 렌즈 판정 결과 + 레이어 A·B·C 수치
출력: "이 매장이 정리 우선이 된 이유" / "조건부인 이유" 2~3줄 설명
```

| 상황 | Claude 역할 |
|---|---|
| 매장 상세 패널 | 3개 렌즈 판정 이유 자연어 설명 |
| 시뮬레이션 결과 | "폐점 시 예상 시나리오 및 주의사항" 요약 |
| 권역 리포트 | "이 권역의 매장 구조에서 주목할 패턴" 인사이트 |

결론을 내리는 역할이 아니라 **"왜 이 매장이 주목받는가"를 사람이 이해할 수 있도록 번역**하는 역할.

---

### 4-5. 방법론 ↔ 목적함수 매핑

분석에 사용하는 방법론과 의사결정 기준(목적함수) 간의 연결 관계.

| 방법론 | 무엇을 계산하는가 | 어느 목적함수에 쓰이는가 |
|---|---|---|
| DBSCAN 클러스터링 | 반경 내 SKT 매장 밀집도 | 커버리지 — 대체 매장이 충분한가 |
| Isolation Forest | 손익·LTV·운영비 이탈 패턴 | 수익성 — 평균 대비 적자·이탈 여부 |
| Huff 모델 | 폐점 시 인근 매장 수요 흡수 확률 | M/S 방어 — 닫아도 고객이 SKT 안에 남는가 |

방법론은 계산 도구이고 목적함수는 판단 기준. 하나의 방법론 출력이 여러 목적함수에 쓰이는 경우도 있음 (예: DBSCAN은 커버리지 판정 외에 Huff 계산의 입력도 됨).

## 6. 화면 구성 (고수준)

### 메뉴 구조
```
매장수 최적화
├── 최적화 현황판   ← AI 스코어카드 리스트 (본사 진입점)
├── 상권 지도       ← 매장 + 경쟁사 오버레이 (현장 진입점)
└── 최적화 시뮬레이터 ← 매장 선택 → 3렌즈 분석 + What-if
```

### 데이터 플로우
```
사내 DB (실적/LTV/비용/상권)
        +
경쟁사 데이터 (SKT/KT/LGU+ 위치)
        ↓
   스코어 엔진
        ↓
  Claude API (설명 생성)
        ↓
  대시보드 (리스트 + 지도 + 시뮬레이터)
```

## 7. 개발 우선순위 (MVP → 고도화)

### Phase 1 — MVP
- [x] 경쟁사 크롤러 (SKT/KT/LGU+ 매장 위치 수집 완료)
- [ ] DBSCAN 클러스터링 (과밀 구역 탐지)
- [ ] Isolation Forest (패턴 이탈 매장 탐지)
- [ ] 상권 지도 (클러스터 시각화 + SKT/경쟁사 오버레이)
- [ ] 주목 매장 리스트 (anomaly score 순)

### Phase 2 — 시뮬레이터
- [ ] Huff 모델 (폐점 시 수요 이동 시뮬레이션)
- [ ] Claude API 연동 (패턴 이탈 이유 + 시뮬레이션 해석 자연어 생성)
- [ ] 매장 상세 패널 (4개 레이어 결과 통합 표시)

### Phase 3 — 고도화
- [ ] Autoencoder (복합 패턴 이상치 탐지 고도화)
- [ ] XGBoost Feature Importance (변수 중요도 도출)
- [ ] 권역 리포트 자동 생성
- [ ] 월별 변화 추이 트래킹

### 업계가 검증한 방법론 (우리 설계에 반영)

**① 잠식(Cannibalization) 분석이 네트워크 재설계의 핵심**
인근 SKT 매장끼리 수요를 나눠 먹는 정도를 수치화해야 함.
매장 하나를 독립 평가하는 것은 불충분 — 인근 매장과의 관계를 반드시 함께 봐야 함. *(arxiv 공간경제학 논문 검증)*
→ **L1 DBSCAN + L3 Huff 모델**에 반영

**② 커버리지 공백 + 잠식 + 화이트존 3개를 동시 분석**
- 커버리지 공백: 우리가 없는 상권
- 잠식: 자사 매장끼리 겹치는 상권
- 화이트존: 경쟁사도 없고 우리도 없는 미개척 상권
→ 지도에 3개 레이어로 시각화하면 "줄일 곳 / 늘릴 곳"이 직관적으로 드러남 *(업계 표준 방법론)*
→ **L3 시뮬레이션 출력 + 상권 지도 레이어**에 반영

**③ SKT 자체 선례: CV센터 8곳 폐점 (2023)**
상담 전용 단일 기능 매장을 전부 폐점하고 판매+상담 통합 직영대리점으로 전환.
→ **"어떤 기능을 하는 매장인가"** 가 폐점 판단의 중요 기준임을 SKT 스스로 실증. *(복수 언론 확인)*
→ **L2 피처 설계**에 채널 기능 유형 포함

**④ 글로벌 통신사 트렌드: 전체 매장 줄이되 기능별 양극화**
미국 통신사: 후불 매장 소폭 감소, 선불(저관여) 매장 증가.
고가 서비스 채널은 통폐합, 저관여 채널은 확산하는 양극화 전략. *(Wave7 Research, 2025)*
→ **L2 이상치 해석 기준**에 채널 유형 분류 병행 적용

### 우리 설계에 추가 반영 사항

| 리서치 발견 | 반영 레이어 | 반영 방법 |
|---|---|---|
| 잠식 분석 | L1 DBSCAN + L3 Huff | 반경 내 SKT 매장 수요 중복률 계산 |
| 채널 기능 분류 | L2 Isolation Forest | 판매+상담 통합 여부 피처로 포함 |
| 화이트존 | L3 시뮬레이션 출력 | 폐점 후 화이트존 발생 여부 시각화 |
| 면적당 효율 KPI | L2 피처 | 유동인구 대비 실계약 전환율 추적 |
