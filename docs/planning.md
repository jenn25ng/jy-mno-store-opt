# 매장수 최적화 서비스 기획서

> SKT 오프라인 매장 포트폴리오 최적화 — 분석 방법론 및 서비스 설계

---

## 1. 문제 정의

### 배경
- SKT 오프라인 매장을 최적화 해서 운영하는 방향으로 전략 전환 필요
- 단순 비용 절감이 아닌 **M/S 방어 + 수익성 개선 + 커버리지 유지**를 동시에 달성해야 함
- 의사결정을 데이터/AI 기반으로 전환하고자 함

### 핵심 질문
> **"어떤 매장을 닫아도 SKT 점유율과 고객 경험에 타격이 없는가?"**

### 목적함수 (단일 정답 없음)
매장 폐점 결정은 단일 지표로 환원 불가. 아래 3가지 렌즈를 동시에 평가:

| 렌즈 | 핵심 질문 | 판단 기준 |
|---|---|---|
| **M/S 방어** | 닫아도 시장점유율이 유지되는가? | 인근 SKT 매장으로 가입 흡수 가능 여부 |
| **수익성** | 운영비 대비 창출 가치가 적자인가? | 비용 대비 LTV·순증 기여도 |
| **커버리지** | 해당 상권의 대체 접점이 있는가? | 반경 내 대체 매장 수·유동인구 커버 여부 |

3개 렌즈 모두 "닫아도 된다" → **폐점 유력 후보**
1~2개 걸림 → **조건부 / 유지 권고**

## 2. 사용자 및 요구사항

### 사용자
| 구분 | 역할 | 주요 관심사 |
|---|---|---|
| 유통혁신팀 | 전국 포트폴리오 조정 결정 | 전국 현황 한눈에, 폐점 후보 목록, 시뮬레이션 결과 |
| 지역본부 | 권역 내 매장 효율 관리 | 내 권역 지도 보기, 특정 매장 상세 분석 |

### 핵심 요구사항
1. AI가 전국 매장에 대해 **폐점 위험도 스코어**를 자동 산출하고 순위를 보여줄 것
2. 특정 매장 선택 시 **3개 렌즈별 평가 + AI 종합 의견** 제공
3. **"이 매장을 닫으면?"** 시뮬레이션: M/S 변화·커버리지 공백·흡수 가능 매장 자동 계산
4. 경쟁사(KT/LGU+) 매장 위치를 지도에 오버레이하여 상권 공백 파악
5. 본사와 현장이 각자 편한 진입점(리스트 vs 지도)으로 접근 가능

## 4. 분석 방법론

### 4-0. 설계 철학

> **"AI는 신호를 주고, 사람이 판단한다"**

매장 폐점에는 정해진 정답이 없음. 인구수 대비, LTV 관점, 경쟁 대응 관점 등 어느 하나로 환원되지 않으며, 전체 최적화보다 **"여긴 줄여야 / 여긴 생겨야"의 지역별 신호 합**으로 접근하는 것이 현실적 방향.

따라서 이 시스템의 설계 원칙:
- **알고리즘이 "폐점하라"고 결론 내리지 않음**
- 각 분석 레이어가 서로 다른 관점의 신호를 제공하고
- 사람(유통혁신팀, 지역본부)이 신호를 종합해 최종 판단

가중 스코어링(weighted scoring) 방식은 직관적이나 가중치 자체가 임의값이라는 근본적 한계가 있어 **주된 방법론으로 채택하지 않음.** 대신 각 레이어가 독립적인 관점을 제공하는 구조로 설계.

> **리서치 연계** → [참고] ①②항 (잠식 분석, 커버리지 3개 레이어 분석)에서 "단일 스코어가 아닌 다차원 신호"의 중요성 확인

---

### 4-1. 레이어 1 — 지리적 밀집 구역 탐지 (DBSCAN 클러스터링) <sup>[1]</sup>

**목적**: "어디가 과밀인가"를 데이터가 스스로 찾도록 함

DBSCAN(Density-Based Spatial Clustering of Applications with Noise)은 인근에 매장이 밀집된 구역을 자동으로 군집화. 사람이 기준을 정하지 않아도 데이터 분포에서 과밀 구역이 드러남.

```
입력: 전국 SKT 매장 위경도
출력: 클러스터 그룹 (밀집 구역) + 고립 매장 (단독 입지)

예시:
  클러스터 A: 수원시 팔달구 내 SKT 매장 5개 → 과밀 구역
  고립 매장:  강원 인제군 SKT 매장 1개 → 단독 입지, 폐점 불가
```

**이 레이어가 주는 신호**: "이 구역은 밀집되어 있어 중복 가능성 있음"
**이 레이어가 주지 않는 것**: 어느 매장을 닫을지 (→ 레이어 2가 담당)

**파라미터**:
- 반경(ε): 도심 0.8km / 수도권 1.5km / 지방 3.0km (상권 유형별 차등)
- 최소 매장 수(min_samples): 3개 이상이면 클러스터로 인정

> **리서치 연계** → [참고] ①항 (잠식 분석이 핵심) — 클러스터 내 매장들이 서로의 수요를 잠식하는 구역을 공간적으로 탐지하는 첫 단계

---

### 4-2. 레이어 2 — 패턴 이탈 매장 탐지 (Isolation Forest) <sup>[2]</sup>

**목적**: "평균적인 매장 패턴에서 얼마나 벗어났는가"를 측정

Isolation Forest는 비지도학습 이상치 탐지 알고리즘. "폐점해야 한다"는 정답 레이블 없이, 전체 매장 중 패턴이 다른 매장을 탐지.

```
입력 피처 (매장별):
  - 월 순증 / 고객 평균 LTV / 월 운영비
  - 인근 SKT 매장 수 / 인근 경쟁사 매장 수
  - 상권 유동인구 / 채널 기능 유형 / M/S

출력: 매장별 anomaly score (낮을수록 더 이상)
```

**중요한 해석 원칙**: 이상치 ≠ 즉시 폐점 후보
- 이상치는 "주목해야 할 매장"이며 이유는 다양할 수 있음:

| 이상치 유형 | 실제 의미 | 판단 방향 |
|---|---|---|
| 운영비 높음 + 순증 낮음 + 인근 SKT 多 | 잠식 받는 비효율 매장 | 폐점 검토 |
| 경쟁사 밀집 속 유일한 SKT 매장 | 전략적 요충지 | 유지 필수 |
| 출점 3개월 미만 | 데이터 미성숙 | 판단 보류 |
| 도서·산간 단독 입지 | 커버리지 유지 목적 | 유지 필수 |

**이 레이어가 주는 신호**: "이 매장은 평균과 다름 — 왜 다른지 들여다볼 것"
**이 레이어가 주지 않는 것**: 다르다고 해서 닫아야 한다는 결론

> **리서치 연계** → [참고] ②항 (커버리지 공백·잠식·화이트존 동시 분석) — 이상치 유형 분류 시 "단독 입지" vs "잠식 받는 과밀 내 매장" 구분에 직결

---

### 4-3. 레이어 3 — 폐점 후 영향 시뮬레이션 (Huff 모델) <sup>[3]</sup>

**목적**: "이 매장을 닫으면 실제로 어떻게 되는가" 검증

레이어 1·2에서 주목 매장이 식별되면, Huff 모델로 폐점 시 수요 이동을 확률적으로 예측.

```
Huff 모델 원리:
P(고객 i → 매장 j) = (매장규모_j / 거리_ij²) ÷ Σ(매장규모_k / 거리_ik²)

→ 특정 매장 제거 시 해당 매장 고객이 인근 SKT 매장으로 이동할 확률 계산
→ 경쟁사 매장으로 이탈할 확률도 추정
```

**시뮬레이션 출력**:
1. **M/S 변화**: 폐점 후 해당 상권 SKT M/S 예상 변동폭
2. **흡수율**: 인근 SKT 매장이 수요를 흡수하는 비율
3. **이탈 위험**: 경쟁사로 넘어갈 것으로 추정되는 고객 수
4. **커버리지 공백**: 폐점 후 SKT 접점이 없어지는 유동인구 규모
5. **연간 비용 절감**: 운영비 × 12

**이 레이어가 주는 신호**: "닫았을 때 숫자로 어떤 일이 벌어지는가"
**이 레이어가 주지 않는 것**: 닫아야 한다/말아야 한다는 결론

> **리서치 연계** → [참고] ①항 (잠식 분석이 핵심) — Huff 모델은 잠식 분석의 확률적 버전으로, "폐점 후 수요가 자사 또는 경쟁사로 얼마나 이동하는가"를 수치화

---

### 4-4. 레이어 4 — 자연어 설명 (Claude API)

**목적**: 수치 결과를 의사결정자가 읽을 수 있는 언어로 변환

```
입력: 레이어 1~3의 분석 결과 (클러스터 위치, anomaly score, Huff 시뮬레이션 수치)
출력: "이 매장이 주목 대상이 된 이유"와 "닫을 경우 예상 시나리오" 자연어 요약
```

**활용 포인트**:

| 상황 | Claude 역할 |
|---|---|
| 매장 상세 패널 | "이 매장이 패턴 이탈로 탐지된 이유" 2~3줄 설명 |
| 시뮬레이션 결과 | "폐점 시 예상 시나리오 및 주의사항" 요약 |
| 권역 리포트 | "이 권역의 매장 구조에서 주목할 패턴" 인사이트 |

Claude API는 결론을 내리는 역할이 아닌 **"왜 이 매장이 주목받는가"를 사람이 이해할 수 있도록 번역**하는 역할.

### 4-5. 전체 파이프라인

```
사내 DB (순증/LTV/운영비/M/S/상권)
           +
경쟁사 크롤링 (KT 257개·LGU+ 1,982개 위치)
           ↓
┌─────────────────────────────────┐
│  레이어 1: DBSCAN               │
│  → 과밀 클러스터 탐지            │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  레이어 2: Isolation Forest     │
│  → 클러스터 내 패턴 이탈 매장    │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  레이어 3: Huff 모델            │
│  → 폐점 시 M/S·커버리지 변화    │
└────────────┬────────────────────┘
             ↓
┌─────────────────────────────────┐
│  레이어 4: Claude API           │
│  → 자연어 설명 + 인사이트        │
└────────────┬────────────────────┘
             ↓
      사람이 최종 판단
  (유통혁신팀 / 지역본부장)
```

### 4-7. 방법론 주석

<sup>[1]</sup> **DBSCAN**: Ester et al., "A Density-Based Algorithm for Discovering Clusters in Large Spatial Databases with Noise", KDD 1996. 공간 데이터의 밀도 기반 클러스터링 표준 알고리즘. scikit-learn 구현체 사용.

<sup>[2]</sup> **Isolation Forest**: Liu et al., "Isolation Forest", IEEE ICDM 2008. 정답 레이블 없이 이상치를 탐지하는 앙상블 방법. 랜덤 분할 트리로 이상 관측값을 빠르게 격리. scikit-learn 구현체 사용.

<sup>[3]</sup> **Huff 모델**: Huff, David L., "Defining and Estimating a Trade Area", Journal of Marketing 28, 1964. 소매 매장 입지 분석의 고전 공간 중력 모델. P(i→j) = (Sj/dij²) / Σ(Sk/dik²). 현재 GIS 기반 입지 분석의 표준 기법.

<sup>[4]</sup> **Autoencoder 이상탐지**: Sakurada & Yairi, "Anomaly Detection Using Autoencoders with Nonlinear Dimensionality Reduction", MLSDA 2014.

<sup>[5]</sup> **XGBoost**: Chen & Guestrin, "XGBoost: A Scalable Tree Boosting System", KDD 2016. Feature importance로 실제 데이터에서 변수 영향도 측정 가능.

<sup>[6]</sup> **Location-Allocation**: ReVelle & Swain, "Central Facilities Location", Geographical Analysis 1970. "P개의 시설로 최대 수요를 커버하는 최적 위치"를 구하는 최적화 모델.

## 6. 화면 구성 (고수준)

### 메뉴 구조
```
매장수 최적화
├── 최적화 현황판   ← AI 스코어카드 리스트 (본사 진입점)
```

## 7. 개발 우선순위 (MVP → 고도화)

### Phase 1 — MVP
- [ ] 경쟁사 크롤러 (KT/LGU+ 매장 위치 수집) ← **KT 257개·LGU+ 1,982개 수집 완료**
- [ ] DBSCAN 클러스터링 (과밀 구역 탐지) ← **시뮬레이션 완료**

## [참고] 외부 리서치 핵심 사항

> 상세 내용: `docs/research-retail-network-optimization.md`

### 업계가 검증한 방법론 (설계에 반영된 내용)

**① 잠식(Cannibalization) 분석이 네트워크 재설계의 핵심** → *레이어 1 DBSCAN + 레이어 3 Huff 모델에 반영*

인근 SKT 매장끼리 수요를 나눠 먹는 정도를 수치화해야 함.
매장 하나를 독립 평가하는 것은 불충분 — 인근 매장과의 관계를 반드시 함께 봐야 함. *(arxiv 공간경제학 논문 검증)*

→ **반영**: DBSCAN으로 잠식 구역을 군집화하고, Huff 모델로 폐점 시 수요 재배분 확률을 계산하는 구조로 설계

**② 커버리지 공백 + 잠식 + 화이트존 3개를 동시 분석** → *4-0 설계철학 + 레이어 2 이상탐지 유형 분류에 반영*

- 커버리지 공백: 우리가 없는 상권
- 잠식: 자사 매장끼리 겹치는 상권
- 화이트존: 경쟁사도 없고 우리도 없는 미개척 상권
→ 지도에 3개 레이어로 시각화 *(업계 표준 방법론)*

→ **반영**: 이상치 유형 분류표의 "전략적 요충지(유지 필수)" / "잠식 받는 비효율 매장(폐점 검토)" 구분에 직접 반영

**③ SKT 자체 선례: CV센터 8곳 폐점 (2023)** → *1. 문제 정의의 배경으로 활용*

상담 전용 단일 기능 매장을 전부 폐점하고 판매+상담 통합 직영대리점으로 전환.
→ **"어떤 기능을 하는 매장인가"** 가 폐점 판단의 중요 기준임을 SKT 스스로 실증. *(복수 언론 확인)*

→ **반영**: 4-6 고도화 방향에서 채널 기능 분류 신호 추가 제안에 근거로 활용. 기능 단일화 매장이 이상탐지에서 먼저 포착될 가능성 높음.

**④ 글로벌 통신사 트렌드: 전체 매장 줄이되 기능별 양극화** → *4-6 고도화 방향에 반영*

미국 통신사: 후불 매장 소폭 감소, 선불(저관여) 매장 증가.
고가 서비스 채널은 통폐합, 저관여 채널은 확산하는 양극화 전략. *(Wave7 Research, 2025)*

→ **반영**: 매장 유형별 분리 분석 방향 및 채널 기능 플래그(판매+상담 통합 여부) 추가에 근거
