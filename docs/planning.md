# 매장수 최적화 서비스 기획서

> SKT 오프라인 매장 포트폴리오 최적화 — 분석 방법론 및 서비스 설계

---

## 1. 문제 정의

### 배경
- SKT 오프라인 매장 수가 과잉 상태. 총 매장 수를 줄이는 방향으로 전략 전환 필요
- 단순 비용 절감이 아닌 **M/S 방어 + 수익성 개선 + 커버리지 유지**를 동시에 달성해야 함
- 의사결정이 감(感)과 경험 기반 → 데이터/AI 기반으로 전환 필요

### 핵심 질문
> **"어떤 매장을 닫아도 SKT 점유율과 고객 경험에 타격이 없는가?"**

### 목적함수 (단일 정답 없음)
매장 폐점 결정은 단일 지표로 환원 불가. 아래 3가지 렌즈를 동시에 평가:

| 렌즈 | 핵심 질문 | 판단 기준 |
|---|---|---|
| **M/S 방어** | 닫아도 시장점유율이 유지되는가? | 인근 SKT 매장으로 가입 흡수 가능 여부 |
| **수익성** | 운영비 대비 창출 가치가 적자인가? | 비용 대비 LTV·순증 기여도 |
| **커버리지** | 해당 상권의 대체 접점이 있는가? | 반경 내 대체 매장 수·유동인구 커버 여부 |

3개 렌즈 모두 "닫아도 된다" → **폐점 유력 후보**
1~2개 걸림 → **조건부 / 유지 권고**

## 2. 사용자 및 요구사항

### 사용자
| 구분 | 역할 | 주요 관심사 |
|---|---|---|
| 본사 전략팀 | 전국 포트폴리오 조정 결정 | 전국 현황 한눈에, 폐점 후보 목록, 시뮬레이션 결과 |
| 영업본부장/팀장 | 권역 내 매장 효율 관리 | 내 권역 지도 보기, 특정 매장 상세 분석 |

### 핵심 요구사항
1. AI가 전국 매장에 대해 **폐점 위험도 스코어**를 자동 산출하고 순위를 보여줄 것
2. 특정 매장 선택 시 **3개 렌즈별 평가 + AI 종합 의견** 제공
3. **"이 매장을 닫으면?"** 시뮬레이션: M/S 변화·커버리지 공백·흡수 가능 매장 자동 계산
4. 경쟁사(KT/LGU+) 매장 위치를 지도에 오버레이하여 상권 공백 파악
5. 본사와 현장이 각자 편한 진입점(리스트 vs 지도)으로 접근 가능

## 3. 데이터 현황

### 3-1. 사내 데이터 (접근 가능)

| 데이터 | 내용 | 활용 렌즈 |
|---|---|---|
| 매장 기본정보 | 위치(위경도), 유형(직영/대리점/판매점), 운영 상태 | 공통 |
| 가입 실적 | 신규/해지/순증 건수 (매장별 월별) | M/S, 수익성 |
| 매장 M/S | 해당 상권 내 SKT 점유율 | M/S |
| 고객 LTV | 매장 유치 고객의 생애 가치 | 수익성 |
| 운영 비용 | 매장별 임대료·인건비·운영비 | 수익성 |
| 유동인구 | SKT 통신 데이터 기반 상권 유동인구 | 커버리지 |
| 상권 데이터 | 사내 보유 상권 분류·분석 정보 | 커버리지 |

### 3-2. 외부 데이터 (수집 필요)

| 데이터 | 수집 방법 | 활용 렌즈 |
|---|---|---|
| KT 매장 위치 | KT 홈페이지 크롤링 (`kt.com/매장찾기`) | M/S, 커버리지 |
| LGU+ 매장 위치 | LGU+ 홈페이지 크롤링 | M/S, 커버리지 |

**크롤링 설계:**
- 수집 항목: 매장명, 주소, 위경도(geocoding), 매장유형
- 수집 주기: 월 1회 (매장 변동 반영)
- 저장: DB 또는 정적 JSON (Polaris `/tmp` 활용)
- 주의: robots.txt 준수, 과도한 요청 방지 (delay 설정)

## 4. 분석 방법론

### 4-1. 복합 스코어 산출

매장별 **폐점 적합도 스코어 (0~100)** 산출. 높을수록 폐점 후보.

```
폐점 스코어 = w1 × M/S방어점수 + w2 × 수익성점수 + w3 × 커버리지점수
```

**M/S 방어 점수:**
- 반경 X km 내 타 SKT 매장 수
- 인근 매장들의 평균 가입 여유 용량
- 해당 매장 폐점 시 예상 M/S 이탈률 (회귀 모델)

**수익성 점수:**
- 월 운영비 / (순증 × 고객 평균 LTV)
- 비율이 높을수록(= 적자 심할수록) 폐점 점수 올라감
- 기준: 동일 유형 매장 평균 대비 하위 N%

**커버리지 점수:**
- 반경 내 대체 SKT 매장 유무
- 인근 경쟁사(KT/LGU+) 매장 수 (많을수록 점수 낮아짐 = 폐점 신중)
- 해당 상권 유동인구 규모

**가중치 (초기값, 조정 가능):**
| 렌즈 | 초기 가중치 |
|---|---|
| M/S 방어 | 40% |
| 수익성 | 35% |
| 커버리지 | 25% |

### 4-2. 폐점 시뮬레이션 로직

특정 매장 선택 시 자동 계산:

1. **M/S 변화 예측**: 해당 매장의 월 가입 건수 → 인근 SKT 매장 흡수율 적용 → 잔여 M/S 계산
2. **커버리지 공백**: 해당 매장의 서비스 반경 → 공백 구역 면적 및 유동인구 규모
3. **대체 매장 추천**: 공백을 흡수 가능한 인근 SKT 매장 Top 3
4. **비용 절감 추정**: 월 운영비 × 12 = 연간 절감액

## 5. AI 도입 계획

### AI가 하는 일

| 단계 | AI 역할 | 기술 |
|---|---|---|
| 스코어링 | 복합 지표 가중 합산 + 이상치 보정 | Rule-based + 통계 모델 |
| 후보 추천 | 폐점 우선순위 Top N 선정 및 이유 설명 | Claude API (자연어 설명) |
| 시뮬레이션 | 폐점 후 M/S·커버리지 변화 예측 | 회귀/공간 분석 모델 |
| 인사이트 | "이 권역에서 X개 줄이면 최적 조합은?" | Claude API (최적화 추론) |

### Claude API 활용 포인트

**1. 폐점 후보 설명 생성**
```
입력: 매장 스코어 데이터 (3개 렌즈 점수, 인근 매장 현황, LTV, 비용)
출력: "이 매장을 폐점 후보로 추천하는 이유" 자연어 설명 (2-3줄)
```

**2. 시뮬레이션 해석**
```
입력: 시뮬레이션 수치 결과
출력: "이 매장을 닫았을 때 예상 시나리오" 요약 및 주의사항
```

**3. 권역 최적화 추론**
```
입력: 권역 내 전체 매장 스코어 + "N개 줄이고 싶다" 조건
출력: 최적 폐점 조합 추천 + 근거
```

## 6. 화면 구성 (고수준)

### 메뉴 구조
```
매장수 최적화
├── 최적화 현황판   ← AI 스코어카드 리스트 (본사 진입점)
├── 상권 지도       ← 매장 + 경쟁사 오버레이 (현장 진입점)
└── 폐점 시뮬레이터 ← 매장 선택 → 3렌즈 분석 + What-if
```

### 데이터 플로우
```
사내 DB (실적/LTV/비용/상권)
        +
경쟁사 크롤링 (KT/LGU+ 위치)
        ↓
   스코어 엔진
        ↓
  Claude API (설명 생성)
        ↓
  대시보드 (리스트 + 지도 + 시뮬레이터)
```
