## 2. 판정 구조 재정의 — 레벨 명시 (F1)

3개 렌즈를 나란히 두지 않고 **관점 레벨**로 묶는다.

### 관점 A · 내부(자사 실적) — "이 매장이 문제인가" (진단·트리거)
- **수익성** (Isolation Forest): 손익·순증·LTV·운영비·자사 M/S
- 데이터: 사내 확정 → **신뢰도 높음**

### 관점 B · 외부(시장·경쟁·지역) — "닫아도 되는가" (영향 검증·가드레일)
- **M/S 방어** (§3의 잔존 예측 모델): 인근 흡수 / 이탈
- **커버리지** (DBSCAN): 상권 내 대체 접점
- 데이터: 경쟁사·행동데이터 → **신뢰도·표기를 내부와 차등**

**현행 → 변경안**

| | 현행 | 변경안(v2) |
|---|---|---|
| 구조 | 3개 렌즈 평면 나열 | 내부(진단) / 외부(검증) 2관점 |
| 신뢰도 | 구분 없음 | 내부=확정 근거 · 외부=참고 신호로 차등 표기 |
| 판정 로직 | 통과 렌즈 수 집계(그대로 유지 = A안) | (동일) — 표현만 2관점으로. 로직 변경은 별도 논의 |

> 판정 로직 자체(수익성을 게이트로 둘지)는 이번엔 **문서 재구성만(A안)**. 로직 변경은 추후.

## 3. 레이어 C 재정의 — Huff 추정 → 행동데이터 기반 잔존 예측 모델 (F2)

### 현행의 한계
- 폐점 후 이동을 **거리²·매장규모 가정**으로 추정 (Huff)
- 잔존율 **0.75 하드코딩**, 경쟁사 규모 `20` 추정, **월 순증**을 배분(고객 base 아님)

### 변경안 — 실측 정답 + 행동 피처로 예측
고객 단위 데이터 확보 가능(확인됨) → 추정이 아닌 **검증 가능한 예측 모델**.

- **단위**: 폐점 후보 매장의 고객 1명
- **정답(학습·검증)**: 이미 닫힌 매장 고객의 1년 내 결과 — **다른 SKT 매장서 기변/유지=잔존(1)** vs **MNP-out=이탈(0)**
- **입력 피처(폐점 전 관측 가능)**:
  - 단골 매장 등록: 인근 다른 매장 단골 등록 여부·대상·거리
  - 교차 거래 이력: 인근 매장서 신규·기변 경험
  - 인근 SKT 매장 활동량(신규·기변 실적)·거리, 경쟁사 밀도
  - 고객 속성: 가입기간·LTV·요금제 등
- **출력**:
  1. 잔존 확률 → 매장별 평균 = **흡수율** → M/S 방어 렌즈 입력
  2. 이동 예상 매장 → 상세 패널 "고객 이동"
- **모델**: 설명 가능한 것 우선(로지스틱 회귀 / GBM + 피처 중요도). 블랙박스 지양.
- **검증**: 닫힌 매장 실측으로 정확도 리포트 → "실제 폐점 사례에서 n% 적중"

> 효과: 하드코딩 0.75·거리² 가정 제거. 사내 실측(사후)을 **닫기 전 예측**으로 확장.

## 4. 실행 권한 레벨 — 투자매장 필터 (F3)

분석(닫아도 되나)과 **별개 축** = 실행 권한(닫게 할 수 있나).

- **투자매장만 폐점 푸시 가능** → 액션 리스트 = **정리 유력 ∩ 투자매장**
- 비투자 정리유력 매장 = "정보성(직접 푸시 불가)"로 분리 표기
- 데이터: 사내 매장 마스터의 투자/비투자 플래그
- 대시보드: 리스트/지도에 투자매장 필터 + 배지

---

## 5. 리밸런싱 프레임 — 정리 ↔ 신설 (F4)

같은 레이어를 **반대 방향**으로 돌려 신설 후보도 탐지.

| 방향 | 신호 | 레이어 |
|---|---|---|
| **정리**(줄일 곳) | 과밀 + 적자 + 흡수 가능 | 현행 3렌즈 |
| **신설**(늘릴 곳) | 커버리지 공백 · 화이트존(SKT·경쟁사 얇음) · 미커버 수요 | DBSCAN 공백 · Huff 반대(신설 시 흡수 추정) |

- 프레임: "매장 닫아라"가 아니라 **"이 권역은 이렇게 재배치하면 효율↑"**
- 신설 쪽은 **수요·화이트존 추정이 추가 필요** → 스코프 큼 → Phase 분리

## 6. 데이터 요구사항 — 샘플 → 실데이터 (F5)

현재 **위치만 실제**, 성능값은 전부 시뮬레이션(`random.gauss`).

| 데이터 | 현재 | 필요(실데이터) |
|---|---|---|
| 매장 위치 | ✅ 실제(SKT/KT/LGU+) | 유지 |
| 순증·LTV·손익·운영비·자사 M/S | ❌ 시뮬레이션 | **사내 실적 연동** |
| 단골 등록(고객 단위) | ❌ 없음 | **신규 연동** (F2 모델용) |
| 신규·기변 실적(고객 단위) | ❌ 없음 | **신규 연동** (F2 모델용) |
| 닫힌 매장 1년 내 기변/MNP-out | ❌ 없음 | **신규 연동** (F2 정답지) |
| 투자매장 플래그 | ❌ 없음 | **신규 연동** (F3 필터) |

- 연동 경로 후보: 사내 **Data Gateway(Athena)** — 접근 가능 테이블 확인 필요
- 고객 단위 모델링은 **프라이버시·거버넌스 승인** 전제
